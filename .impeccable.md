# Impeccable — Design Context

This file captures the design context for TravelTwin so every design/polish
session starts grounded. Verify against code before relying on specifics.

## Design Context

### Users
Digital travelers in Romania and Europe who want to plan a complete trip —
flight, hotel, rental car, day-by-day itinerary, maps, weather — in one
integrated flow, without juggling 8+ apps (Skyscanner, Booking, Maps,
TripAdvisor, ChatGPT…). They answer a short wizard and get three ready-to-book
packages (Economic / Balanced / Premium) with live prices and an AI itinerary.
Native RO/EN; first-class Romanian market support (OTP/CLJ/TSR/IAS/SBZ, RON,
full Romanian copy). Context of use: planning leisure travel, often on mobile,
comparing options and trusting live numbers. This is also a licență (bachelor's
thesis) showcase project — it will be demoed and graded.

### Brand Personality
**Confident & effortless.** Three words: *trustworthy, premium, calm.* Acts
like an expert concierge that quietly does the hard work. Voice is clear and
plain (it earns trust by being legible, not loud). Emotional goal: the user
feels *taken care of* and *certain* — never overwhelmed by a fragmented,
multi-tab process. Real data is the hero; the UI gets out of the way.

### Aesthetic Direction
Clean, premium surfaces with restrained, purposeful motion. Vibrant Orange
(`--primary-500: #E85D04`) as the confident accent against Dark Navy
(`--secondary-500: #1A1A2E`) and tinted neutrals. Both **light and dark** modes
are first-class (dark mode uses true-black `#000` surfaces with inverted
primary ramp). Typography: Inter for display + body, JetBrains Mono for code —
hierarchy via the `.text-display/.text-h1…` scale in `globals.css`. Let live
travel imagery and real prices carry the visual weight; chrome stays quiet.

- **Anti-references**: generic AI-slop dashboards; cyan-on-dark / purple-blue
  gradient hero metrics; everything-in-a-card grids; bouncy/elastic motion;
  loud gradient text on numbers. Avoid making prices feel like marketing fluff —
  they must read as trustworthy data.

### Design Principles
1. **Data is the hero, chrome is quiet.** Prices, durations, and itineraries
   get the visual weight; surfaces, borders, and accents stay restrained.
2. **Confidence through legibility.** Clear hierarchy, generous spacing, honest
   contrast (WCAG AA). Trust is earned by being easy to read, not flashy.
3. **Motion with purpose, never decoration.** Use the existing ease-out
   entrances/staggers for state changes and page loads; respect
   `prefers-reduced-motion`. No bounce, no elastic.
4. **One accent, used decisively.** Orange marks the primary action and live
   status — not every surface. Hierarchy comes from restraint.
5. **Effortless on every device.** Mobile-first, ≥44px touch targets, no
   amputated functionality on small screens; both themes equally finished.

### Quality bar
Flagship / portfolio-grade — sweat the details: pixel alignment, complete
interaction states, consistent spacing tokens, smooth 60fps motion.

### Technical constraints
Next.js 16 (App Router, Turbopack), React 19, Tailwind v4 (class-based dark
mode via `@custom-variant`), Framer Motion, next-intl (RO/EN), next-themes.
TypeScript strict — never `any`. In JSX never use HTML entities (`&apos;` etc.)
— use curly braces or plain text. Design tokens live in
`src/app/globals.css`; reuse them rather than hard-coding values.
